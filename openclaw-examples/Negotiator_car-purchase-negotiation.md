# 汽车购买自动谈判代表 (Car Purchase Negotiation)

## Sources
- https://aaronstuyvenberg.com/posts/clawd-bought-a-car

## 1. 应用场景 (Application Scenario)
买车过程通常极其繁琐且充满信息不对称，涉及与多个经销商的反复博弈和讨价还价。普通购车者往往缺乏时间和精力去逐一获取报价并进行压价。在此案例中，用户需要购买特定配置（特定外观和内饰颜色）的 2026 款 Hyundai Palisade Hybrid，但希望免去亲自沟通的痛苦。

OpenClaw 在此充当了**谈判代表（Negotiator）**这一全新系统架构角色，代理用户与多个外部实体（汽车经销商）并发进行多轮邮件沟通和价格博弈，以期达到利益最大化（即最低购车价格）。

## 2. 技术方案 (Technical Architecture/Solution)

该方案结合了信息爬取、表单自动化和长周期邮件监控来进行持续的谈判博弈。

- **前期价格发现与目标设定**：使用浏览器 / 搜索插件（如访问 Reddit 相关板块）来分析目标区域内的真实历史成交价，从而让 Agent 建立一个基准谈判底价（例如 $57,000）。
- **自动化库存检索与线索生成**：
  - 使用 Browser Automation 访问专门的库存查询网站（如 hexorcism），锁定 50 英里范围内符合车辆颜色和型号的 VIN 码。
  - 自动进入各经销商官网，将用户的真实姓名、邮箱和电话填入咨询表单，索取首轮“出门价”（out-the-door price）。
- **多方竞价策略与定时监控 (Cron/Heartbeat)**：
  - 设定 `Cron Job`，每隔几分钟自动轮询检查 Gmail 邮箱。
  - 当收到经销商发来的 PDF 报价单后，提取核心价格，并通过 Email 工具自动将最低报价单（或截图）转发给其他竞争经销商，要求他们提供更低的价格（Price Matching / Beating）。
- **人机协同控制 (Human-in-the-loop)**：
  - 为了防止 Agent 自行决定关键财务条款，配置了“在发送重要回复前提示用户”的 Hook。过滤掉贷款利率、以旧换新等复杂选项，强制 Agent 仅就裸车总价进行沟通。

## 3. 实现效果 (Results/Outcomes)

- **显著的经济与时间收益**：OpenClaw 最终成功制造了经销商之间的竞价战，获得了一笔 $4,200 的经销商折扣，将最终成交价降至 $56,000，超出预定目标，且用户基本在后台“挂机”即可。
- **存在的不足与风险**：由于涉及多线程上下文，Agent 曾出现过上下文混淆（将拒绝其他人的回复错误地发给了正在谈判的经销商）。这表明在执行多方并发谈判时，对 Session 和 Thread 的严格隔离至关重要；此外，全自动化暴露了未经确认可能泄露敏感信息的风险。

## 4. 其他相关信息 (Other Info)
这是 OpenClaw （曾用名 Clawdbot / Moltbot）引发病毒式传播的核心案例之一。由于该任务需要连续几天、7x24 小时不间断地监控邮件和回复，展示了 OpenClaw 作为持久化后台进程（而非单次对话 Chatbot）的强大潜力。作者为此甚至专门配置了一台独立运行的 Mac Mini 节点（Node）。
